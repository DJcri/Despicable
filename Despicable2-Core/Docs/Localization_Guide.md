# Despicable 2 (Core) Localization Guide

## Folder layout

- `Languages/English/Keyed/`  
  Keyed translations referenced from C# via `"Key".Translate(...)`.

- `Languages/English/DefInjected/<DefType>/`  
  Def field translations (labels, descriptions, reportStrings, etc.).


## Split strategy (Core vs NSFW)

Core contains:
- All translations required by **Core code** (including *safe shared UI* like the Manual Interaction menu keys).

NSFW contains:
- Only NSFW-specific keys and DefInjected strings.

Rule of thumb:
- If Core calls it (or can call it), it must live in Core.
- If it only exists when NSFW is enabled, it can live in NSFW.

## Adding a new language

1. Copy `Languages/English/` to `Languages/<YourLanguage>/`.
2. Translate the XML values (do not change the element names).
3. For DefInjected files, keep `DefName.field` element names unchanged.

## Guardrails (Build-Time Audit)

This mod runs a localization audit **after build** to prevent missing translations and keep DefInjected mirrors in sync.

- Script: `Tools/LocalizationAudit.ps1`
- If you see a build failure listing missing keys, add them to `Languages/English/Keyed/*.xml`.
- DefInjected: missing English entries are auto-generated into `Languages/English/DefInjected/<DefType>/AutoGenerated_DefInjected.xml`.

To temporarily disable (not recommended), set MSBuild property:

`D2SkipLocalizationAudit=true`

> Note: `CodeStrings_UI.xml` is a normal Keyed file (not a TODO list). It contains UI strings extracted from C#.

# Localization Audit Guardrails

This mod includes a build-time localization audit that prevents "raw key" UI and keeps **English DefInjected** mirrors in sync with Def text.

## What it checks
- **Keyed coverage:** every `"SomeKey".Translate(...)` used in C# must exist in `Languages/English/Keyed/*.xml`

- **DefInjected orphan cleanup:** prunes orphan nodes (defName no longer exists) from `AutoGenerated_DefInjected.xml`, and errors on orphans in other DefInjected files (override with `Tools/DefInjectedKeep.txt`)
- **DefInjected coverage:** for every Def that has `label`, `description`, `reportString`, etc, an **English** DefInjected entry exists (auto-generated if missing)
- **Unused Keyed cleanup:** unused English Keyed entries are pruned so translators only see strings the code actually uses

If a key is looked up dynamically (so the audit regex can't see it), add it to:

`Tools/LocalizationKeepKeys.txt`

## How it runs
Both Core and NSFW projects call the script automatically after a successful build in Visual Studio (Windows).

If you need to skip the audit temporarily, set this MSBuild property:

`D2SkipLocalizationAudit=true`

## Manual run
PowerShell:

```powershell
powershell -ExecutionPolicy Bypass -NoProfile -File .\Tools\LocalizationAudit.ps1 -ModRoot .\
```

Exit codes:
- `0` OK
- `1` missing keyed entries referenced by code (fix required)
- `2` script setup error


## Extra guardrail: UI literal warnings
The audit also emits **warnings** if it sees likely player-facing UI calls that contain a raw string literal (e.g. `Widgets.Label("...")`).

- Fix: wrap the text in `.Translate()`
- Suppress a specific line (rare): add `// loc-ignore` to that line

## Extra guardrail: translation string safety
The audit **fails the build** if English Keyed/DefInjected strings contain patterns that cause RimWorld to warn:

`Translation data for language English has N errors`

Specifically:
- Forbids ASCII navigation arrows like `->` and raw angle bracket characters (`<` or `>`) inside translation strings.
  - Use `→` or `/` separators instead.
- Errors on duplicate Keyed or DefInjected keys across files.
- Errors on malformed `{0}` placeholders or unbalanced braces.

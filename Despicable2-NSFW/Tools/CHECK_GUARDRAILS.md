# Guardrail Checker

Use this checker after a feature pass, before packaging a release, or any time you want a quick "did I just plant future debt?" sanity scan.

## What it checks

- empty `catch { }` blocks
- likely mutable `static` fields
- shared static collections
- oversized source files
- suspicious generic names for caches/registries/holders
- committed generated folders like `.vs` or `obj`

## What it is for

This tool is intentionally lightweight and conservative.

It is **not** a full linter and it does **not** replace judgment. It exists to catch the most common drift patterns:

- convenience globals
- settings mirrored into runtime state
- file-blob growth
- vague naming that hides ownership
- committed build clutter

Some findings will be acceptable. The point is to force a deliberate review before the repo quietly grows barnacles.

## How to run

From the repository root:

```bash
python Tools/check_guardrails.py
```

Optional flags:

```bash
python Tools/check_guardrails.py --max-file-lines 450 --warn-only
```

## Exit behavior

- By default, findings return a non-zero exit code.
- With `--warn-only`, the checker prints findings but exits successfully.

## Suggested use

Run it when:

- finishing a larger feature
- adding new caches or registries
- touching global/runtime state
- doing a cleanup pass
- preparing a release zip


## Localization guardrails (new)

The checker now also covers localization drift:

- Missing **Keyed** entries for any `"Key".Translate(...)` used in C# (ERROR).
- Heuristic warnings for likely unlocalized UI literals in calls like `Widgets.*("...")` (REVIEW).  
  Suppress a specific line with `// loc-ignore`.
- Missing **English DefInjected** entries for Def text fields (WARN by default).
- Unused **English Keyed** entries (WARN by default). Auto-prune is available.

To auto-generate missing English DefInjected scaffolding:

```bash
python Tools/check_guardrails.py --fix-definjected

To auto-prune unused English Keyed entries:

```bash
python Tools/check_guardrails.py --prune-unused-keyed
```

If a key is used dynamically (so the audit regex can't see it), list it in:

`Tools/LocalizationKeepKeys.txt`
```

This writes missing entries into:

`Languages/English/DefInjected/<DefType>/AutoGenerated_DefInjected.xml`


## DefInjected orphan guardrail
`check_guardrails.py` will fail if it finds DefInjected nodes whose `defName` no longer exists in `Defs/` (outside of `AutoGenerated_DefInjected.xml`).

- Auto-generated files are pruned automatically.
- For intentional keep cases, add wildcard patterns to `Tools/DefInjectedKeep.txt`.


## UI literal protection

- The guardrails now scan wrapper-based UI APIs such as `D2Widgets.*`, `D2Text.*`, `D2Section.*`, `D2Tabs.*`, `D2ActionBar.Item(...)`, `Dialog_MessageBox.*`, `Dialog_TextEntrySimple(...)`, and `NextTextBlock(...)` in addition to direct `Widgets.*`/`FloatMenuOption(...)` calls.
- Use `// loc-ignore: reason` for an intentional player-facing exception and `// loc-allow-internal: reason` for internal/debug-only literals.
- Release builds can opt into strict mode with `D2FailOnUiLiterals=true`, which upgrades likely unlocalized UI literals from warnings/review findings to build-blocking errors.
- Prefer keyed wrapper overloads where available, such as `ButtonTextKey`, `LabelKey`, `LabelClippedKey`, `DrawWrappedLabelKey`, `DrawCaptionStripKey`, `ItemKey`, and the keyed tab helpers.

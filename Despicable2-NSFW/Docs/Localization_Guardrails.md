# Localization Guardrails

These guardrails keep localization stable while you iterate on UI and Defs.

## Rules of thumb

### C# (UI, tooltips, messages)
- Never ship player-facing English literals in code.
- Use `"Key".Translate(...)` and define the key in `Languages/English/Keyed/*.xml`.
- Prefer stable semantic keys (recommended), but any key works as long as it's defined.

### Defs (labels/descriptions/reportStrings)
- Def text lives in `Defs/`.
- The **English DefInjected mirror** is kept up to date for translation scaffolding.
- Missing English DefInjected entries are auto-generated on build into:  
  `Languages/English/DefInjected/<DefType>/AutoGenerated_DefInjected.xml`

## Automatic audit on build
After building in Visual Studio, the project runs:

`Tools/LocalizationAudit.ps1`

It will:
- Fail the build if any `.Translate()` key is missing from English Keyed.
- Auto-add missing English DefInjected entries based on `Defs/`.

To temporarily disable (not recommended), set MSBuild property:

`D2SkipLocalizationAudit=true`


## Heuristic warnings
The audit prints warnings for likely unlocalized UI literals (e.g. `Widgets.*("literal")`).

- Preferred fix: wrap the text in `.Translate()` and add a Keyed entry.
- Rare suppression: add `// loc-ignore` on that line.


## DefInjected orphan cleanup
The audit also checks English `DefInjected` for **orphan nodes** (entries whose `defName` no longer exists in `Defs/`).

- Orphans in `AutoGenerated_DefInjected.xml` are **pruned automatically**.
- Orphans in other DefInjected files are treated as an **error** (to prevent shipping dead translations).
- If you intentionally keep an orphan for an optional patch, add a wildcard rule to `Tools/DefInjectedKeep.txt`.

## Translation string safety
RimWorld's translation loader is sensitive to certain characters in Keyed/DefInjected text.

- Avoid ASCII navigation arrows like `->` and avoid raw angle-bracket characters (`<` or `>`).
  - Use Unicode arrows `→` or `/` separators instead.
- Keep placeholders well-formed: `{0}`, `{1}`, with balanced braces. Use `{{` / `}}` for literal braces.
- Do not duplicate Keyed or DefInjected keys across files (duplicates can trigger translation load errors).


## UI literal protection

- The guardrails now scan wrapper-based UI APIs such as `D2Widgets.*`, `D2Text.*`, `D2Section.*`, `D2Tabs.*`, `D2ActionBar.Item(...)`, `Dialog_MessageBox.*`, `Dialog_TextEntrySimple(...)`, and `NextTextBlock(...)` in addition to direct `Widgets.*`/`FloatMenuOption(...)` calls.
- Use `// loc-ignore: reason` for an intentional player-facing exception and `// loc-allow-internal: reason` for internal/debug-only literals.
- Release builds can opt into strict mode with `D2FailOnUiLiterals=true`, which upgrades likely unlocalized UI literals from warnings/review findings to build-blocking errors.
- Prefer keyed wrapper overloads where available, such as `ButtonTextKey`, `LabelKey`, `LabelClippedKey`, `DrawWrappedLabelKey`, `DrawCaptionStripKey`, `ItemKey`, and the keyed tab helpers.
